## Passage 7: How the Internet Works

**Theme: Web & Networking**

When you type a **URL** in your **browser**, you are acting as a **client**. You send a **request** across the network to a **server** far away. The server then sends a **header** and the **content** back to you as a **response**.

This conversation follows a **protocol** called **HTTP**. If the server cannot find the page, it might show an **error**. Your computer uses a **cache** to save some data locally. This way, the next time you visit the same site, the **loading** speed is much faster. The internet is like a huge post office delivering millions of digital letters every second!

**Questions:**

1. In the text, who or what is the "client"?
    
2. What does the server send back to the client?
    
3. What is "HTTP" according to the passage?
    
4. How does a "cache" help the user?
    
5. What happens if the server cannot find the page you requested?

## Passage 9: The World of Open Source

**Theme: Tools & Collaboration (Git)**

Have you ever heard of "Open Source"? It means the **source code** is free for everyone to see and change. Developers use a tool called Git to manage their **repository**.

If you want to help, you can **clone** the project to your computer. After you make some changes, you **commit** your code with a small **message**. Then, you send a **pull request** to the owner. If the owner likes your work, they will **merge** your code into the main **branch**. This is how thousands of people around the world **collaborate** to build great software like Linux or Android.
